spec-driven development

Cinco fases. Uma de cada vez, sem pular.

Em vez de pular direto ao código, cada feature maior passa por fases sequenciais, e cada fase produz um artefato que alimenta a próxima. As decisões ficam explícitas e verificáveis antes de virar código.

0BRAINSTORM
/brainstorm

Explorar a ideia

Reduzir incerteza antes de fixar requisitos. Onde a feature ainda é hipótese.

1DEFINE
/define

Capturar requisitos

O quê e os critérios de aceite: quando exatamente está "pronto".

2DESIGN
/design

Definir arquitetura

A spec técnica: o como construir, antes de escrever a primeira linha.

3BUILD
/build

Implementar com verificação

Código real + lint/testes + relatório de build. A qualidade é provada, não assumida.

4SHIP
/ship

Encerrar e aprender

Arquivar a feature e registrar as lições, o acervo que alimenta a KB.

↳ Tarefa pequena e direta? Pula o ciclo completo e usa o atalho /dev, o Dev Loop.

O coração é o .claude/ que cada projeto recebe.

Commands
Slash commands auto-contidos, uma fase/ação por comando. .claude/commands/
Rules
Contexto sempre ativo: workflow SDD, CLI-first, docs-first, taxonomia da KB. .claude/rules/
Agents
9 experts de papel universais; os de domínio surgem na curadoria. .claude/agents/
KB
Base de conhecimento em 4 camadas: começa vazia e cresce com o projeto. .claude/kb/
Hooks
curation-nudge (staleness, read-only) + guards de segurança. .claude/hooks/
AGENTS.md
Contrato canônico p/ qualquer agente; o CLAUDE.md aponta para ele. raiz do projeto
as 4 camadas da KB
business/Qual é a regra de negócio / métrica?
tools/Como funciona esta tecnologia em geral?
implementation/O que nós construímos / configuramos?
operations/Como rodo, reinicio ou recupero?

A camada tools/ puxa documentação atual via context7 MCP.

Universais, não de tecnologia. Cada um com escopo distinto, invocado quando a tarefa se beneficia de contexto próprio.

Cada agente declara role e connects_to: um grafo de quem encadeia com quem (ex.: debugger → test-writer). Todos rodam em model: inherit; agentes de domínio (ex.: sql-reviewer) são gerados sob demanda pela curadoria.

/max · potência total, defesas intactas.

Para um bloco de trabalho denso ou autônomo: lê todo o contexto do projeto, recomenda o modelo maior, e atua como orquestrador-mestre delegando a experts.

  • Lê KB, memória, skills, MCPs e hooks; o orçamento de tokens fica em silêncio.
  • Recomenda modelo + effort maiores, sem trocar à força (segue model: inherit).
  • Reduz a fricção dos prompts de permissão do não-crítico.

/peers · várias sessões, um só projeto.

Rodando dois chats no mesmo projeto (duas features, dois orquestradores)? As sessões se enxergam por um quadro file-based: sem daemon, sem rede.

  • Cada sessão publica a presença sozinha (branch, summary derivado sem rede e sem dados pessoais, heartbeat) via o hook peer-heartbeat.
  • /peers mostra quem está ativo e em que área, e entrega recados assíncronos entre sessões.
  • Aparece no /status; sob /max, o orquestrador-mestre consulta os peers antes do fan-out. Dois mestres se coordenam, não se atropelam.

Seis princípios inegociáveis.

01

Genérico e context-free

O scaffold não carrega contexto de tarefa: base limpa que se especializa quando inicializada.

02

Auto-otimização na init

A curadoria ajusta agentes, treina a KB e sincroniza o contexto. Sob demanda, não pré-fabricada.

03

SDD como espinha

Features grandes passam pelas 5 fases; tarefas pequenas usam o Dev Loop.

04

Claude-first, portável

Otimizado para Claude Code, com artefatos estruturados para portar a outras ferramentas.

05

Qualidade verificável

Nada é "pronto" sem lint, testes e critérios de aceite. Não inventar dados; não versionar segredos.

06

Segurança em camadas

Rede hook-based, não prompt-based. Educar, não barrar: prioriza o risco irreversível, deixa o resto fluido.

quickstart

Três blocos e você está rodando.

PowerShell · quickstart
# 0. clonar o repositório
$ git clone github.com/Phenrique-DataDev/Native-sdd
# 1. preparar a máquina + ~/.claude
$ onboarding/install.ps1
# 2. criar projeto novo já com o scaffold
$ New-SddProject meu-projeto -Open
# 3. dentro do projeto (no Claude Code)
/setup contexto do projeto
/init cura agentes + treina KB
✓ comece o dia por /status
